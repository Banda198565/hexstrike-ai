## 1. Назначение

Samson SBM (Simulation / Purple-Team) — on-prem платформа для:

1. ingest целей из desktop/container pool;
2. OSINT/Web3 enrichment (Shodan, FOFA, Arkham) с бюджетными лимитами;
3. active financial / LLM injection emulation против in-scope HTTP;
4. RAG explainability (PostgreSQL + pgvector);
5. guardrail reverse-proxy (Round-2 assert `drop` / `hitl`);
6. synthetic Web3 purple-team на **локальном Anvil** (sweeper / drainer families).

Точка входа:

```text
python3 samson/orchestrator.py <command>
```

Docker: `ENTRYPOINT ["python3", "/workspace/samson/orchestrator.py"]`.

## 2. Архитектура ядра (слои)

```text
Operator / CLI / docker compose
        │
        ▼
Continuous Audit & Ingestion
  target_loader.py + orchestrator.py
        │
   ┌────┼────────────────┐
   ▼    ▼                ▼
Payload  Async Emulation  Recon clients
Registry (httpx)          (Shodan∥Arkham∥FOFA)
        │
        ▼
PostgreSQL 15 + pgvector (embeddings vector(768))
        │
        ▼
RAG Oracle (retrieve / brief / report)
        │ on breach
        ▼
Guardrail Compiler + Proxy :8787 → Round-2 assert → close()
```

| Слой | Путь | Роль |
| --- | --- | --- |
| Data & Schema | `samson/redteam/schemas.py` | Pydantic-контракты |
| Ingestion | `samson/core/target_loader.py` | Desktop/container → `IngestedTargetPool` |
| Core runtime | `samson/core/` | config, DB, scope, payloads, HTTP |
| Orchestration | `samson/orchestrator.py` | CLI + audit loops |
| Pipeline | `samson/pipeline/` | continuous audit helpers |
| Emulation | `samson/redteam/adversary_executor.py` | scoped async HTTP |
| RAG | `samson/rag/` | ingest / retrieve / rerank / oracle |
| Guardrail | `samson/redteam/guardrail/` | IBAN, HITL, aiohttp proxy |
| Recon | shodan / arkham / hybrid_recon | cache-first OSINT |
| Web3 sandbox | `validation_node.py`, purple-team modules | Anvil-only drills |
| Persistence | `samson/migrations/001…009_*.sql` | schema |
| Packaging | `Dockerfile`, `docker-compose.yml` | on-prem stack |

## 4. Red-team / purple-team подсистемы

| Модуль | Команда CLI | Режим |
| --- | --- | --- |
| Continuous audit | `run-continuous-audit` | payloads → emulate → guardrail → proxy verify |
| Bulk audit | `run-bulk-audit` | pool → Shodan∥Arkham → continuous → matrix |
| Shodan | `shodan-lookup` | cache → rate → budget → live |
| Arkham | `arkham-lookup` | address intel → `web3_recon_artifacts` / balance gate |
| FOFA | `fofa-lookup`, `fofa-hunt-redis` | hybrid Redis:6379 hunt |
| Sweeper purple-team | `sweeper-purple-team` | Anvil full-balance drain → defense `is_risk` |
| Drainer purple-team | `drainer-purple-team` | EVM/USDT Anvil approve+transferFrom; TRX = defense IOC |
| Validation node | (внутри bulk/Web3) | `LocalBlockchainSandbox` — только local chain |
| Redis resilience | (executor) | unauth AI-context exposure drill |
| Garak / PyRIT / ATLAS | `garak-scan`, `pyrit-eval` | model health / risk / mapping |
| Financial sandbox | guardrail deploy / proxy | IBAN firewall + HITL |

**Политика Web3:** attack-пути sweeper/drainer только Anvil/Hardhat (`31337`/`1337`). Mainnet/BSC/TRON drain malware не интегрируется.

## 11. Политика безопасности (обязательная)

- Секреты только в env (`SAMSON_*`), не в git.
- Scope + operator enforcement через `ScopeEnforcer`.
- Shodan/FOFA: cache-first, rate-limit, credit reserve.
- Arkham balance gate перед Web3 signer (local Anvil bypass).
- Web3 attack drills: **только** local RPC/chain; mainnet/production chain IDs hard-refuse.
- Live drain-боты (EVM/TRX/USDT malware) **не** являются operational модулями системы.
